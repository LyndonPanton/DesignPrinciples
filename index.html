Welcome

As a software developer tasked with creating and managing large projects it can be easy to over complicate code or repeat yourself so other coders have made principles that can help prevent these problems.

KISS

The KISS (Keep It Simple Stupid) design principle was coined by Kelly Simpson and states that most code works best when it is kept simple and unnecessary complexity should be avoided.

DRY

The DRY (Don't Repeat Yourself) design principle was coined by Andres Hunt and David Thomas and states that you should avoid writing the same code multiple times and instead use syntax such as loops, functions and class to avoid that. It is the opposite of the WET (Write Everything Twice / We Enjoy Typing) solution.

YAGNI

The YAGNI (You Aren't Gonna Need It) design principle was coined by Ron Jeffries and states that when writing code do not added functionality to a program just because you may need it, because you probably aren't.
